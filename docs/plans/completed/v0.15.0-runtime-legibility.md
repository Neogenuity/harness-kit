# Runtime legibility

Status: **completed** — v0.15.0 (released 2026-07-14)

## Objective

Make the *running application* operable and observable by an agent: boot per
worktree, seed deterministic data, expose logs/health, and ship a
live-verification skill — so behavioral confidence stops ending at green
unit tests.

## Value

Green unit tests are a weak behavioral sensor for UI, distributed-system,
and performance work. Current practice closes the loop against the live
app: OpenAI made their app bootable per git worktree with an ephemeral local
observability stack (logs/metrics/traces) per instance
(<https://openai.com/index/harness-engineering/>, validated 2026-07-10 via
InfoQ 2026-02); Anthropic's evaluator drives the page itself — navigating,
screenshotting, checking endpoints and database state
(<https://www.anthropic.com/engineering/harness-design-long-running-apps>).
The kit discovers build commands at init but nothing makes the app
*runnable* by an agent.

## Scope

1. **The `dev.sh` contract** (convention, not a generic script template): a
   convention specifying `dev.sh up|health|seed|down`, a one-object JSON v1
   response, idempotent lifecycle semantics, deterministic explicit seeding,
   and worktree ownership. Init authors and integrity-pins the tailored script
   only for detected application repos; existing installs opt in. The shipped
   mechanism is `scripts/harness/lib/dev-instance.sh`, with `suffix [namespace]` and
   `port <base> <span> [namespace]` commands derived from the physical Git
   worktree root. *Acceptance: the skill can rely on the contract without
   repo-specific guessing; helper regressions and manifest/guard coverage are
   deterministic.*
2. **Live-verification skill**: `templates/docs/skills/verify-live/SKILL.md`
   — the reproduce → observe → change → re-run loop; drive the affected
   flow, not just tests; browser/screenshot steps for UI surfaces; where to
   find logs/traces for the booted instance.
3. **Matrix: browser tooling pointers** — a stamped, surface-aware note on
   native browser options and Playwright CLI/MCP, re-verified against primary
   docs per ADR 004. Native desktop/IDE support must never be presented as a
   headless-CLI guarantee; the installed skill stays self-contained rather
   than linking back into the plugin cache.
4. **verify.sh example**: a commented `full_gate "smoke" bash scripts/dev.sh health`
   pattern showing where live checks slot into the gate order.
5. **init/update/audit integration**: recon proposes boot, health, seed, port,
   log, and trace commands from Compose/Procfile/manifest scripts; init asks
   only unresolved details and conditionally authors the contract, convention,
   skill, AGENTS links, and generated stubs. Update/audit offer existing apps
   opt-in adoption. Audit calls only the read-only `health` action and
   distinguishes missing, drifted, stopped, unhealthy, and ready states.
6. **Fixture app + behavioral evidence**: a root-only Python HTTP app exercises
   two linked-worktree instances, deterministic seed data, health JSON, logs,
   ownership-safe teardown, and cleanup. A positive `verify-live-runtime`
   capability task must record at least 2/3 passes on Codex
   `gpt-5.6-luna`; screenshot/browser execution is documented but non-gating
   because the headless eval runner provisions no browser.

## Out of scope

Shipping any observability stack (compose files for Grafana etc. — point,
don't vendor); performance benchmarking; provider browser-tool
reimplementation or automatic installation; a devcontainer; mathematical
port-collision guarantees (finite hashing yields a candidate and `up` must
fail loudly rather than reuse another instance's port).

## Dependencies

The behavioral-evals runner (used to exercise the contract end-to-end on
the fixture app). The fixture app itself is this plan's own scope item 6 —
no earlier plan provides it.

## Verification

- **Deterministic runtime fixture:**
  `bash docs/evals/tasks/verify-live-runtime/test-fixture.sh` passes. It runs
  main and linked-worktree instances concurrently on distinct suffixes and
  candidate ports, proves deterministic reseeding and valid lifecycle JSON,
  checks HTTP health/content and retained logs, stops A while B stays healthy,
  exercises partial-startup cleanup, and leaves no listener behind after a
  readiness timeout.
- **Behavioral cell:** Codex `gpt-5.6-luna` passed **3/3**
  (`pass@k=1`, `pass^k=1`, rate `1.00`) under run
  `20260714-170246-codex-gpt-5.6-luna`; recorded in
  `docs/evals/baselines.json`. Browser/screenshots were non-gating as planned.
- **Cross-review:** Terra found no actionable issue in Luna's content. Luna
  found two fixture issues (readiness-timeout orphaning and a theoretical
  candidate-port collision assertion); Terra confirmed both lead fixes and
  the focused fixture rerun.
- **Integrated gates:** `bash scripts/harness/verify` passed after integration and
  again after the task-scoped eval-runner network fix, including shellcheck,
  manifests, every template test, eval grader validity, the runtime fixture,
  and harness coherence. The release commit receives one final full run.

## Decisions

- 2026-07-14 — **Eval network access is explicit and task-scoped:** TASK.md
  metadata defaults to `network: none`; `required` adds Codex's
  `sandbox_workspace_write.network_access=true` only for that task. Live eval
  fixtures remain self-contained and may not depend on external services.
- 2026-07-14 — **Delegation and single writers:** Terra owns shipped script
  mechanism plus fixture/eval evidence; Luna owns shipped content/provider
  guidance. Both branch from this activation commit. The lead alone owns this
  plan, root dogfood copies, generated root stubs, version/changelog, and
  `.harness-manifest`; cross-review swaps their implementation areas before
  final integration.
- 2026-07-14 — **Conditional + pinned adoption:** application repos get an
  authored `scripts/dev.sh` plus convention/skill/stubs at init; non-app repos
  report N/A; existing installs opt in. `dev.sh` is project policy, manifest-
  pinned and diff-only, never a generic script template.
- 2026-07-14 — **Explicit lifecycle:** `up|health|seed|down`; every valid action
  emits one JSON v1 object. `up` never seeds, `health` is read-only, `seed`
  resets known data, and `down` owns only the current worktree.
- 2026-07-14 — **Evidence floor:** deterministic two-worktree HTTP coverage is
  CI-gated; Codex `gpt-5.6-luna` runs 3 trials and must pass at least 2. Browser
  screenshots remain non-gating until the eval runner provisions that surface.
- 2026-07-10 — Contract over stack, and contract over template: the kit
  standardizes the `dev.sh` interface and ships only the worktree-name
  helper as mechanism — never the infrastructure, never an all-TAILOR
  script body.
