# v0.24.0 — Content/IA migration: the committed `.harness/` layer and docs IA

Status: **completed** (v0.24.0, 2026-07-22) — Phase 4 of
[../standard-consumer-layout.md](../standard-consumer-layout.md)

## Objective

Give the committed `.harness/` layer its content: `policies/` (agent policy
docs), `templates/` (plan/ADR/PR doc templates), `schemas/` (machine
contracts), `evals/` (scenarios/fixtures/rubrics/baselines), and `agents/`
(canonical personas). Reshape the human docs IA (`standards/`, `runbooks/`,
`product/`, `generated/`, `references/`, `PLANS.md`, root `ARCHITECTURE.md`).
Move canonical skills to `.agents/skills/` (the cross-provider standard —
Codex reads it natively; stubs stay generated for the rest). Ship `GEMINI.md`
and `.github/copilot-instructions.md` thin-pointer templates.

## Value

Completes the layout ADR 010 started: after this phase every zone of the
standard tree has its final contents, so Phase 5 (generated adapters) writes
into a stable structure and Phase 6's fresh-repo bootstrap demos the real
end state. The `.agents/skills/` move is the trend-setting piece — canonical
content at the emerging vendor-neutral location instead of a kit-private one.
