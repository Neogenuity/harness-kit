# v0.23.0 — Mechanism re-home: the `scripts/harness/` command surface

Status: **completed** (v0.23.0, 2026-07-22) — Phase 3 of
[../standard-consumer-layout.md](../standard-consumer-layout.md)

## Objective

Move the kit-owned mechanism into `scripts/harness/` behind extensionless
verb commands (`bootstrap`, `verify`, `sync`, `check-instructions`,
`check-docs`, `detect-drift`, `validate-plan`, `run-evals`) over a shared
`lib/`, split the hooks by ownership (mechanism hooks with config-extracted
policy in `scripts/harness/hooks/`; the project-invariants hook repo-owned at
`.harness/hooks/`), make `verify` a kit-owned runner reading a repo-owned
`.harness/gates.conf`, and move runtime state to gitignored `.harness/var/`.
After this phase, "which files are the kit's" is answerable from the path
alone, and `scripts/harness/` is sync-with-delete.

## Value

The layout is the trend-setting deliverable of the umbrella plan, and this is
its biggest and only-cheap-now move: every path here is referenced by
provider hook configs, check #8d tuple tables, CI templates, and docs — all
shipped from the kit, so contract and consumers change together, exactly
once, before any external adopter exists. The verify runner/config split
completes the mechanism/policy separation: updates stop diffing policy prose
out of shell bodies.
