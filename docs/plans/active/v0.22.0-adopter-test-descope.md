# v0.22.0 — Adopter test descope + ship-contract cleanups

Status: **active** — Phase 2 of
[../standard-consumer-layout.md](../standard-consumer-layout.md); executes
the descope originally queued as
[../adopter-test-descope.md](../adopter-test-descope.md) (now unblocked by
v0.21.0's retirement mechanism).

## Objective

Stop shipping harness-kit's own conformance suites to adopters. The
install/update/recovery suites, the checker suite, the eval-integrity suite,
and the fixture-isolation suite become maintainer-only (root-only,
` # tailored`-pinned, like `test-provider-templates.sh`); adopters keep the
hook behavioral tests, the small runtime suites, and gain a shipped
post-init/post-update smoke test. Fold in the remaining ship-contract
cleanups: `dest=` content entries in the kit-manifest and a #9d refinement.

## Value

Every adopter audit currently re-proves harness-kit's historical upgrade
matrix (~200KB of suites) instead of the adopter's own installed policy — an
ownership-boundary leak the original plan rated cleanup-priority and the
Codex v0.20.0 review seconded. v0.21.0's retirement contract makes the
descope a clean migration instead of an orphan-file hazard.
