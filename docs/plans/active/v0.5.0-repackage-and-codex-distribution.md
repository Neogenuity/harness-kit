# v0.5.0 — Repackage to `plugins/harness-kit/` + Codex distribution

Status: **active**

## Objective

Move the distributed plugin from `plugin/` to `plugins/harness-kit/` and add
a Codex distribution channel, so the same source tree installs as a plugin
in both Claude Code and Codex.

## Value

Codex reads project skills from `.agents/skills/` today, and the kit
installs *into* Codex via the manual clone-and-copy path the README
documents — workable, but not versioned and not updatable in place. Codex
plugins (`.codex-plugin/plugin.json`, marketplaces at
`.agents/plugins/marketplace.json`, plugins conventionally under `plugins/`
— per Codex's plugin docs; re-verify and stamp at execution, see scope
item 8) are the versioned distribution channel, and their layout
conventions match a `plugins/harness-kit/` tree. One repackage buys a second
first-class install path with zero content duplication.

## Scope

1. `git mv plugin plugins/harness-kit` (internal refs are relative — survive).
2. Packaging: `plugins/harness-kit/VERSION` (single version source);
   `.claude-plugin/plugin.json` → 0.5.0; new
   `plugins/harness-kit/.codex-plugin/plugin.json` (name/version/description,
   `"skills": "./skills/"`); new root `.agents/plugins/marketplace.json`
   (local source `./plugins/harness-kit`); root `.claude-plugin/marketplace.json`
   source → `./plugins/harness-kit`.
3. Root tailored mechanism updates: `verify.sh` manifests gate extends to all
   four JSONs plus VERSION == both plugin.json versions;
   `test-template-sync.sh` TPL path; `guard-project-policy.sh` version reads
   from VERSION.
4. CI globs + `.cursor/rules/templates.mdc` paths.
5. Docs sweep (README + Codex install section, llms.txt, AGENTS.md,

6. ADR 007 — dual-provider packaging (both manifests checked in verbatim;

8. Provider matrix: new **Distribution** row — Claude Code
