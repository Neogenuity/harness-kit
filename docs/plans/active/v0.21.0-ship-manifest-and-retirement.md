# v0.21.0 — Declarative ship-manifest + retired-file mechanism

Status: **active** — Phase 1 of
[../standard-consumer-layout.md](../standard-consumer-layout.md)

## Objective

Replace the three hard-coded shipped-set lists in `install-lib.sh` (plus their
duplicate enumeration in `check-harness.sh` check #9c) with one shipped,
integrity-pinned, guard-protected declarative file — `scripts/kit-manifest` —
that states every shipped path's ownership layer, install destination, and
retirement status; and teach update mode to **remove** a retired file when,
and only when, its on-disk copy is pristine and untailored.

## Value

This is the prerequisite every later restructure phase leans on: retirement
is the migration mechanism for all path moves and descopes. It also closes a
shipped defect today: any file the kit stops shipping (v0.20.0's
`test-install.sh` was the live case) orphans a copy in the installed repo and
turns check #9c red, with a by-hand `rm` as the only remedy. Four separate
encodings of "what ships" collapse to one.

## Scope

- [x] `templates/scripts/kit-manifest` (+ root twin): plain-text lines
  `<layer> <path> [dest=<repo-relative>]`, layers
  `mechanism | policy | optional-policy | content`, plus `retired <path>`
  lines. Bash-3.2 `read`-parseable, comment- and blank-tolerant, no jq.
  *Acceptance: `install-lib.sh` derives every file set from it; the three
  `_HARNESS_*` list variables are gone.*

  `harness_persist_base`, `harness_update_decision`, `harness_update_apply`

- [x] ADR 009 (declarative ship-manifest + retirement contract); ADR 005

## Decisions

- 2026-07-22 — **Two manifests, two jobs, two names**: `scripts/kit-manifest`
  is the *ship contract* (checked-in, kit-authored, itself mechanism);
  `scripts/.harness-manifest` stays the *integrity pin* (generated,
  per-install). Merging them was rejected — the ship contract must be
  byte-identical across installs to be updatable, while the integrity pin is
  necessarily per-install.
- 2026-07-22 — **`remove` never fires on drifted or tailored files**, even
  when retired: a retired path with local changes is kept and reported for
  manual review. Deleting user-modified content is the one failure mode this
  mechanism must never have.
