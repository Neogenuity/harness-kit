# Provider Matrix

Where each harness looks for each capability, and which artifact the kit
installs there. "Stub" = generated by `scripts/sync-agent-skills.sh`;
"thin" = hand-written pointer carrying only frontmatter + canonical-source
line. Verify against current harness docs when wiring a provider you haven't
used recently — hook event names in particular are still evolving.

| Capability | Claude Code | Cursor | Codex | OpenCode | `.agents` standard |
| --- | --- | --- | --- | --- | --- |
| Instructions | `CLAUDE.md` (thin → AGENTS.md) | `.cursor/rules/*.mdc` (thin) | `AGENTS.md` (native) | `AGENTS.md` (native) | `AGENTS.md` (native) |
| Skills | `.claude/skills/<slug>/SKILL.md` (stub) | `.cursor/skills/` (stub) | `.codex/skills/` (stub) | `.opencode/skills/` (stub) | `.agents/skills/` (stub) |
| Subagents | `.claude/agents/*.md` (thin) | `.cursor/agents/*.md` (thin) | — | — | — |
| Hooks | `.claude/settings.json` → `hooks` | `.cursor/hooks.json` | — | — | — |
| Permissions | `.claude/settings.json` → `permissions` | (harness UI) | (trust model) | (config) | — |
| MCP servers | `.mcp.json` (project) | `.cursor/mcp.json` | `.codex/config.toml` `[mcp_servers.*]` | `opencode.json` `"mcp"` | — |

## Hook event mapping

All events point at the same portable scripts in `scripts/hooks/`.

| Portable script | Claude Code event | Cursor event | Notes |
| --- | --- | --- | --- |
| `session-context.sh` | `SessionStart` | — (no equivalent yet) | Plain stdout injected into context |
| `guard-secrets.sh` | `PreToolUse` matcher `Read\|Grep` | `beforeReadFile` | Exit 2 = deny in both |
| `format.sh` | `PostToolUse` matcher `Edit\|Write` | `afterFileEdit` | Always exits 0 |
| `guard-project-policy.sh` | `Stop` | `stop` | advise-once protocol (see lib.sh) |

## Payload differences the scripts already absorb

- **File path**: Cursor sends top-level `file_path`; Claude Code nests it as
  `tool_input.file_path` (Read/Edit/Write) or `tool_input.path` (Grep).
  `lib.sh:hook_file_path` tries all three.
- **Stop-loop guard**: Claude Code sends `stop_hook_active` (bool); Cursor
  sends `loop_count` (int). `lib.sh:hook_advise_once` handles both and falls
  back to plain text for anything else.
- **Deny semantics**: exit code 2 + stderr reason denies in both; JSON
  `{"decision":"block","reason":...}` (Claude Code) / `{"followup_message":...}`
  (Cursor) are only needed for the stop-hook continue-the-turn trick.

## Per-provider notes

- **Claude Code** also honors `AGENTS.md` in recent versions, but keep
  `CLAUDE.md` as the thin entry — it's the guaranteed load and carries the
  quality gates.
- **Cursor rules** (`.mdc`) want summarized key points inline, not just a
  pointer — keep the summary short and cite the canonical doc path;
  `check-harness.sh` verifies every cited path exists.
- **Codex / OpenCode** have no hook system; they still get the full docs/
  knowledge base, skills stubs, and AGENTS.md natively. Guards degrade
  gracefully: native permission denies and CI remain.
- **`.agents/skills/`** is the emerging vendor-neutral location; ship stubs
  there so new harnesses that adopt the standard work on day one.
- **Tool-managed files**: some ecosystem tools (e.g. Laravel Boost) write
  their own full-copy skills/rules into provider dirs on install/update.
  After running such a tool, re-run `bash scripts/sync-agent-skills.sh` and
  let `check-harness.sh` confirm the stubs are restored.
